# FCL Management v5.6

Добавлены две независимые системы допуска.

## Допуск в семью

```text
/создать-доступ
```

Выдаёт роль `Призванный` и оформляет никнейм:

```text
Призванный | Static ID
```

## Допуск к Посольству

```text
/создать-доступ-посольство
```

Выдаёт роль `Союзник` и оформляет никнейм:

```text
Союзник | Static ID
```

Обе команды принимают необязательный параметр `баннер`.

## Журналы

- `ACCESS_LOG_CHANNEL_ID` — журнал допуска в семью.
- `DIPLOMACY_LOG_CHANNEL_ID` — журнал дипломатии.
- Если `DIPLOMACY_LOG_CHANNEL_ID` не указан, используется `ACCESS_LOG_CHANNEL_ID`.

## Поведение союзной кнопки

После нажатия бот:

1. Запрашивает Static ID.
2. Проверяет его корректность и уникальность.
3. Проверяет, что пользователь не является членом семьи.
4. Выдаёт роль `Союзник`.
5. Меняет никнейм на `Союзник | Static ID`.
6. Сохраняет событие в базе.
7. Отправляет личное дипломатическое приветствие.
8. Публикует запись в журнале дипломатии.
